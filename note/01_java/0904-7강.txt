1. 상속(inheritance)
상속이란 부모클래스가 가지고 있는 속성과 메서드를 활용할 수 있는 개념

접근제어자 [final/abstract] class 클래스이름 extends 상위클래스(super class) {
	추가할 멤버변수선언;
	생성자;
	추가할 메소드선언;
    }

-상속이란 기존의 클래스를 재사용해서 새로운 클래스를 작성하는 것
-두 클래스를 부모와 자식(조상과 자손)으로 관계를 맺어줄 수 있다.
-자손은 조상의 멤버를 상속받으나, private멤버는 직접 제어할 수 없다.
-자손의 멤버 개수는 조상보다 작을 수 없다(같거나 많다)
-자바에서는 다중 상속이 지원되지 않는다.

2. 상속이 필요한 이유
 -첫째, 기존의 프로그램이 있다면 상속을 통해 결과물을 빠른 시간내에 도출가능 
	또한 대부분의 검증이 잘 되어 있어 버그가 없을 수 도 있음.
 -둘째, 다양한 객체를 상속을 통해서 하나의 객체로 묶을 수 있음.

3. Object클래스
- 조상이 없는 클래스는 자동적으로 상속받게된다.
- 상속계층도의 최상위에 위치하고 모든 클래스는 
  Object클래스에 정의된 11개의 메서드를 상속 받는다. EX)toString(),hashCode()

4.오버라이드의 이해
 - 부모 클래스의 메소드를 자식 클래스에서 재정의 함. 
cf. 오버로딩=인자의 타입이 다르면 같은 이름의 메소드라도 다른 기능으로 중복 정의가 가능한것

5. super
 -하위 클래스에 의해 가려진 상위클래스의 맴버변수나 메소드에 접근할 때
  super.객체변수
  super.메소드이름(매개변수)
  를 첫줄에서 사용하여 오버라이드하여 사용할 수 없는 부모클래스의 메소드를 
  자식 클래스에서사용 할 수 있음.
